üîê 1. Overview & Authentication

API Base URL

https://services.leadconnectorhq.com

Authentication

GHL uses API Keys for authentication. There are two types:

Key Type Format Recommendation
Private Integration Starts with pit- or alphanumeric ‚úÖ Recommended - Long-lived, stable
JWT Token Starts with eyJ... ‚ö†Ô∏è Not Recommended - Expires, must refresh
‚ö†Ô∏è Important: API Key Selection

Always use Private Integration keys for server-to-server integrations. JWT tokens expire and will cause 401 errors if not refreshed.

Required Headers

{
  "Authorization": "Bearer YOUR_API_KEY",
  "Version": "2021-07-28",  // Standard API
  // OR
  "Version": "2021-04-15"   // Objects API (Custom Objects)
}

üìç 2. API Endpoints Reference

Contact Custom Fields

GET/locations/{locationId}/customFields

Returns all custom fields for contacts at a location.

Version: 2021-07-28

‚ö†Ô∏è Limitation

This endpoint does NOT return folder/group information. Fields are returned flat.

Pipelines (Opportunities)

GET/opportunities/pipelines?locationId={locationId}

Returns all pipelines and their stages.

Version: 2021-07-28

‚ÑπÔ∏è Note

The old endpoint /locations/{locationId}/pipelines is deprecated.

Tags

GET/locations/{locationId}/tags

Returns all tags for a location.

Version: 2021-07-28

Custom Objects (Schema)

GET/objects/?locationId={locationId}

Returns all custom object schemas including system objects.

Version: 2021-04-15 (Objects API)

‚úÖ Returns
  • System objects: contact, opportunity, business
  • Custom objects: custom_objects.annuities, etc.

Custom Object Fields

GET/objects/{objectKey}/fields?locationId={locationId}

Returns all fields for a specific custom object.

Version: 2021-04-15

‚ÑπÔ∏è URL Encoding

The objectKey must be URL-encoded. Example: custom_objects.annuities ‚Üí custom_objects.annuities

üì¶ 3. Custom Objects Discovery

The Discovery Journey

Finding the correct Custom Objects API required extensive debugging. Here's what we learned:

What DOESN'T Work:

  • /locations/{locationId}/customValues - Returns "Custom Values" (location settings), NOT Custom Objects
  • /locations/{locationId}/customFields - Only returns Contact fields

What WORKS:

Endpoint Returns
/objects/?locationId=xxx All object schemas (system + custom)
/objects/{key}/fields?locationId=xxx All fields for a specific object

Identifying Product Objects

Custom Objects you create in GHL have keys prefixed with custom_objects.:

// Example custom object keys
custom_objects.annuities
custom_objects.life
custom_objects.medicare
custom_objects.bd_ria

Filtering Product Objects

var targetKeys = [
  'custom_objects.annuities',
  'custom_objects.life',
  'custom_objects.medicare',
  'custom_objects.bd_ria'
];

var productObjects = allObjects.filter(function(o) {
  return o.key && targetKeys.includes(o.key);
});

  • CHECKBOX - Multi-select checkboxes
  • RADIO - Single-select radio buttons
  • MULTIPLE_OPTIONS - Dropdown multi-select
  • TEXTBOX_LIST - List of text inputs
  • SIGNATURE - Signature capture
  • FILE_UPLOAD - File attachment

üìÅ 5. Folder Structure Handling

Contact Fields: No Folder Info

‚ö†Ô∏è GHL API Limitation

The /customFields endpoint does NOT return folder/group information for contact fields. All fields come back as a flat array.

Workaround: Group contact fields by dataType in the UI.

// Group by dataType since no folder info
var byType = {};
fields.forEach(function(f) {
  var type = f.dataType || 'OTHER';
  if (!byType[type]) byType[type] = [];
  byType[type].push(f);
});

Custom Objects: parentId IS the Folder

‚úÖ parentId = Folder ID

Custom Object fields have a parentId property that references the folder/group they belong to.

// Group by parentId for folder structure
var fieldsByFolder = {};
fields.forEach(function(f) {
  var folderId = f.parentId || 'root';
  if (!fieldsByFolder[folderId]) fieldsByFolder[folderId] = [];
  fieldsByFolder[folderId].push(f);
});

// Sort fields within each folder by position
Object.keys(fieldsByFolder).forEach(function(folder) {
  fieldsByFolder[folder].sort(function(a, b) {
    return (a.position || 0) - (b.position || 0);
  });
});
‚ÑπÔ∏è Folder Name Lookup

The parentId is just an ID. To get folder names, you would need a separate API call to fetch folder definitions, which GHL doesn't currently expose publicly.

üíª 6. Code Patterns for Google Apps Script

Secure API Key Storage

// Store API key in Script Properties (not in code!)
function saveGHLSettings(apiKey, locationId) {
  var props = PropertiesService.getScriptProperties();
  props.setProperty('GHL_API_KEY', apiKey);
  props.setProperty('GHL_LOCATION_ID', locationId);
}

function getGHLSettings() {
  var props = PropertiesService.getScriptProperties();
  return {
    apiKey: props.getProperty('GHL_API_KEY'),
    locationId: props.getProperty('GHL_LOCATION_ID')
  };
}

Making API Calls

function makeGHLRequest(endpoint, version) {
  var settings = getGHLSettings();
  var url = 'https://services.leadconnectorhq.com' + endpoint;
  
  var options = {
    method: 'get',
    headers: {
      'Authorization': 'Bearer ' + settings.apiKey,
      'Version': version || '2021-07-28'
    },
    muteHttpExceptions: true
  };
  
  var response = UrlFetchApp.fetch(url, options);
  var code = response.getResponseCode();
  
  if (code !== 200) {
    throw new Error('GHL API returned ' + code + ': ' + response.getContentText());
  }
  
  return JSON.parse(response.getContentText());
}

Fetching Custom Objects with Fields

function getGHLCustomObjects() {
  var settings = getGHLSettings();
  var objectsUrl = '/objects/?locationId=' + settings.locationId;
  
  // Use Objects API version
  var result = makeGHLRequest(objectsUrl, '2021-04-15');
  var objects = result.objects || result;
  
  // Filter for custom objects only
  var customObjects = objects.filter(function(o) {
    return o.key && o.key.indexOf('custom_objects.') === 0;
  });
  
  // Fetch fields for each object
  customObjects.forEach(function(obj) {
    var fieldsUrl = '/objects/' + encodeURIComponent(obj.key) + 
                   '/fields?locationId=' + settings.locationId;
    var fieldsResult = makeGHLRequest(fieldsUrl, '2021-04-15');
    obj.fields = fieldsResult.customFields || fieldsResult.fields || [];
  });
  
  return customObjects;
}

Complete Sync Function

function syncAllGHLData() {
  var settings = getGHLSettings();
  
  var result = {
    customFields: {},
    customObjects: [],
    pipelines: [],
    tags: []
  };
  
  // 1. Contact Custom Fields
  var fieldsResult = makeGHLRequest(
    '/locations/' + settings.locationId + '/customFields'
  );
  result.customFields['All Fields'] = fieldsResult.customFields || [];
  
  // 2. Pipelines
  var pipelinesResult = makeGHLRequest(
    '/opportunities/pipelines?locationId=' + settings.locationId
  );
  result.pipelines = pipelinesResult.pipelines || [];
  
  // 3. Tags
  var tagsResult = makeGHLRequest(
    '/locations/' + settings.locationId + '/tags'
  );
  result.tags = tagsResult.tags || [];
  
  // 4. Custom Objects (with fields)
  result.customObjects = getGHLCustomObjects();
  
  return result;
}

üîß 7. Troubleshooting Guide

401 Invalid JWT

Cause Solution
JWT token expired Use a Private Integration key instead
Wrong API key type Generate new key in GHL Settings ‚Üí Integrations

OAuth 400 Error

Cause Solution
Missing OAuth scopes in appsscript.json Remove explicit oauthScopes array - let GAS auto-detect
Typo in scope name Use script.external_request (singular)

Custom Objects Not Loading

Symptom Cause Solution
Returns "Custom Values" Wrong endpoint Use /objects/ not /customValues
Empty objects array Wrong API version Use version 2021-04-15
Fields missing Need separate call Call /objects/{key}/fields for each

All Fields in "Other" Folder

‚ÑπÔ∏è This is Expected for Contact Fields

The GHL API does not expose folder structure for Contact Custom Fields. This is a known limitation.

Workaround: Group by dataType instead.
