Implementation Roadmap

Note: Shared Services are woven throughout all phases, not a discrete phase.

Phase 1: Foundation

Objective: Organize existing projects and establish folder structure

  • Audit all existing projects - document current state, dependencies, issues
  • Create SuperProject folders: RAPID_TOOLS/, SENTINEL_TOOLS/, PRODASH_TOOLS/
  • Move existing repos into appropriate folders
  • Update Git remotes and any cross-project references
  • Document current MATRIX structure and data locations

Phase 2: Database Architecture

Objective: Create three-matrix structure and migrate data

  • Create RAPID_MATRIX Google Sheet with reference data tabs
  • Create SENTINEL_MATRIX Google Sheet with B2B tabs
  • Create PRODASH_MATRIX Google Sheet with B2C tabs
  • Migrate carrier/product/IMO/comp grid data ‚Üí RAPID_MATRIX
  • Migrate producer/deal/commission data ‚Üí SENTINEL_MATRIX
  • Migrate client/account/activity data ‚Üí PRODASH_MATRIX
  • Validate data integrity after migration
  • Set up Script Properties in all apps to reference new MATRIX IDs

Phase 3: RIIMO Platform

Objective: Build the Operations UI for RAPID shared services

  • RIIMO base app structure (modules + left nav)
  • Data Maintenance pipeline UI
  • Tech Maintenance pipeline UI
  • Security pipeline UI
  • Connect RIIMO to RAPID_MATRIX
  • Module views for each shared service

Phase 4: App Development

Objective: Scope ‚Üí Develop ‚Üí Publish all apps per platform

RIIMO/RAPID Apps:

  • C3 - Complete development and publish
  • TOMIS - Scope, develop, publish
  • Proposal Maker - Scope, develop, publish
  • Contract Generator - Scope, develop, publish
  • LC3 - Scope, develop, publish
  • RAPID Import - Complete development and publish
  • CAM - Complete development and publish
  • MCP-Hub - Complete development and publish

SENTINEL Apps:

  • DAVID-HUB - Complete development and publish
  • sentinel - Complete development and publish

PRODASH Apps:

  • PRODASH - Complete development and publish

Phase 5: Pipeline Implementation

Objective: Build workflow stages in all three platforms

RIIMO Pipelines:

  • Data Maintenance pipeline (Matrix Sync ‚Üí Validation ‚Üí Cleanup ‚Üí Backup)
  • Tech Maintenance pipeline (Deploy ‚Üí Testing ‚Üí Monitoring ‚Üí Hotfix)
  • Security pipeline (Access Control ‚Üí Audit Log ‚Üí Compliance ‚Üí Backup/DR)

SENTINEL Pipelines:

  • Prospecting pipeline (New Lead ‚Üí Engaged ‚Üí Coordinating ‚Üí HUB Initiated)
  • Sales Process pipeline (Report Req'd ‚Üí Analysis ‚Üí Proposal ‚Üí NDA/LOI ‚Üí Close)
  • Transition pipeline (Docs Kit ‚Üí Tech Setup ‚Üí Credentials ‚Üí Data Migration ‚Üí LIVE)

PRODASH Pipelines:

  • Prospecting pipeline (New Lead ‚Üí Engaged ‚Üí Coordinating ‚Üí Scheduled)
  • Sales Process pipeline (Discovery ‚Üí Analysis ‚Üí Proposal ‚Üí Authorization ‚Üí Close)
  • NewBiz pipeline (Application ‚Üí Underwriting ‚Üí Issued ‚Üí Placed ‚Üí Active)

Phase 6: QUE Suite

Objective: Build quoting tools for all product lines

  • Complete QUE-Medicare (existing)
  • Scope QUE-Life quoting module
  • Develop QUE-Life quoting module
  • Publish QUE-Life
  • Scope QUE-Annuity quoting module
  • Develop QUE-Annuity quoting module
  • Publish QUE-Annuity
  • Integrate all QUE modules with Proposal Maker

Phase 7: MDJ Integration

Objective: Set up AI assistants with proper access (final phase)

  • Configure MDJ-BD with SENTINEL_MATRIX access
  • Configure MDJ-Service units with PRODASH_MATRIX access
  • Configure MDJ-Sales units with PRODASH + SENTINEL access
  • Configure MDJ-Executive with all MATRIX access
  • Configure MDJ-Marketing with RAPID_MATRIX (C3) access
  • Test scoped access patterns
  • Deploy MDJs to respective units

Future: Advanced Features

  • RPI Command Center (cross-suite communications)
  • Trinity Data Method Level tracking
  • Cross-platform reporting and analytics
  • QUE Evolution to PostgreSQL (500K+ client scale)
  • B2I Product Ownership (future channel)
  • MDJ Client-Facing (self-service)
