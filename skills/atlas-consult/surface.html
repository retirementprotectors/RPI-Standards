How to invoke

Any warrior or MDJ specialist doing data-import / migration / intake / BoB work

/atlas-consult

Walks through consulting ATLAS's Source Registry, Tool Registry, and Wire Definitions before any new data pipeline work — so work reuses what's already registered instead of recreating it blind.

The 6-step process

1
Before anything else

Read _SOURCE_REGISTRY

Every carrier × product × domain, gap status (RED/YELLOW/GREEN), current vs. target method. Do not discover sources by browsing folders when ATLAS already catalogs them.

2
Before building anything

Read _TOOL_REGISTRY

~150 registered pipeline tools across 6 categories (intake/queuing, extraction/approval, normalization/validation, matching/dedup, external enrichment, bulk operations). Use registered tools — do not reinvent the pipeline.

3
Understand the path

Read WIRE_DEFINITIONS

The established path from source → intake → extraction → approval → MATRIX → frontend. Follow the wires.

4
Even DATA-OUT sessions

Still check ATLAS

The source registry tells you if a data source is already known, its expected method, and which tools handle it — even when working from raw data toward the system.

5
Never eyeball folders

Priority lists reference the gap analysis

Data-work priority lists reference _SOURCE_REGISTRY gap analysis, not a guess from folder contents.

6
Never silently register

Unregistered sources: FLAG, don't auto-register

Found a source not in _SOURCE_REGISTRY? Flag it for JDM with file/folder name, what it appears to contain, and a register/skip/unclear recommendation. Agents do NOT write to _SOURCE_REGISTRY without explicit approval — the registry stays trustworthy only if it stays clean.
