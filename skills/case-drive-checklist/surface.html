The 7-step process

1
Open the case

SWEEP — the 5 Surfaces

Registry · Data · State (Firestore) · Comms (BigQuery recall) · Auth (carrier-direct). Pull carrier-direct wherever possible. Don't start a case cold — its history is one recall away.

2
Carrier = SSOT

RECONCILE the 3 systems of record

Line up every number across Carrier ↔ Firestore ↔ ACF. Flag every drift. The carrier-direct value wins — an on-file/Firestore number is the floor, not today's truth.

3
The deliverable spine

POLISH the Roadmap

Drive the roadmap to current truth; iterate with the writer/JDM to LOCK. It's the live case record. The one-pager + pay-plan distill it — build/sync those after the strategy locks (don't re-sync every iteration).

4
MEGAZORD's lane (VOLTRON flags, read-only)

FIRESTORE True-Up 100% Gate #1

Write every carrier-confirmed fix to Firestore: DOB, account values, rider status, beneficiaries, dedupe. Gate closed = Firestore matches carrier-direct, no stale fields, no duplicates.

5
The document file

ACF True-Up 100% Gate #2

Every doc that should be on file is: carrier statements, illustrations, RMD letters, signed forms (distribution / ACH / activation), beneficiary forms. Pull/file what's missing. Gate closed = nothing referenced that isn't filed.

6
Make it happen

EXECUTE + Document

Run the open work (the forms, the activations), then log the standing instruction in ProDash so it recurs — don't leave it a one-time manual pull.

7
Owner sign-off

SIGN-OFF + Park

JDM eyeballs before anything goes client-facing. Then the case-sub stands by and updates the roadmap as truth moves. Death-gate only on explicit close.

Cadence: reconcile at roadmap-build, and RE-VERIFY at LOCK. A case is "roadmap-100%" once Steps 1–3 are done & locked; it's "case-100%" only when Gates 4 (Firestore) & 5 (ACF) are both closed and Step 6 executed. Track the gate status per case right on the roadmap.
